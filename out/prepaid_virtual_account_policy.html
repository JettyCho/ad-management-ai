v2 2026-05-28 ADM팀 내부문서 협의 필요 항목 포함 SMS 기능 신규 개발 필요

1도입 배경 및 목적

버즈빌 광고센터에 선불충전 도입 시 일반 PG 가상계좌를 활용하려 했으나 PG사가 광고업 특성(비가시적 상품)·고금액·환불 건수 다수를 이유로 거절하였다. 이에 신한은행과 직접 계약하여 벌크 가상계좌를 선발급받아 풀로 운영하는 방식을 채택한다.

현재 수동 프로세스(Sales팀 시트 입력 → BM팀 수동 매칭)를 자동화하여 운영 공수를 제거하고, 대행사 운영 환경에서의 충전·환불·세금계산서 정책을 명확히 정립한다.

3가상계좌 발급 정책

항목내용
발급 권한계정 책임자만 가능
발급 단위광고계정 1개 = 가상계좌 1개
변경 가능 여부변경 불가 (발급 후 고정)
예금주(주)버즈빌 — 신한은행 벌크 발급 시 고정. 정산 담당자 식별 목적으로만 활용
사전 조건계정 책임자의 비즈니스 정보(사업자등록증) 승인 완료
⚠️
가상계좌 번호 유출 리스크

가상계좌 번호가 유출되어도 해당 계좌로 입금이 들어오는 것이므로 금전적 피해는 없다. 단, 잘못 입금된 경우 환불은 등록된 환불 계좌로만 가능하므로 약관에 명시 필요. 잘못 입금 사고 프로토콜은 11절 참조.

4충전 정책

가상계좌에 입금하면 신한은행 실시간 웹훅을 통해 즉시 광고머니로 전환된다. 웹훅 구현 완료

항목내용
충전 주체광고계정 내 누구나 — 계정 책임자, 대행사, 정산 담당자 모두 가능
충전 방법발급된 신한은행 가상계좌로 은행 이체
충전 즉시 반영신한은행 웹훅 → 광고머니 자동 반영
💡
실무 관행

대행사가 운영을 대행하는 경우 일반적으로 대행사가 대행사 자금으로 충전하고 환불도 대행사가 수령한다. 계정 책임자가 직접 충전하는 것도 기술적으로 가능하나, 환불은 등록된 환불 계좌로만 나간다.

5대행권 설정 신규 기능

대행사와 함께 광고계정을 운영하고 선불충전을 사용할 때, 정산에 필요한 정보를 설정하는 기능이다. 세금계산서 수신처 미설정 시 광고 집행은 가능하지만 해당 월 세금계산서를 발행할 수 없다.

①
대행사 계정 초대
대행사 선택 및 광고 운영 권한 위임
②
세금계산서 수신처 설정 필수
계정 책임자 직접 수신 또는 대행사 위임발행 선택
계정 책임자 직접 수신 → 계정 책임자 앞으로 발행
대행사 위임발행 → 대행사 앞으로 발행
③
정산 담당자 지정
계정 책임자 또는 대행사 중 1명 지정 — 정산금액 검토 및 확정 담당
④
환불 계좌 등록 선택
대행사가 신청 → 계정 책임자 승인 → 신한은행 계좌실명조회 인증 (환불 신청 전까지 등록)
📋
세금계산서 수신처 미설정 시 세금계산서 발행 불가

월 말일까지 ①②③이 완료되지 않으면 해당 월의 세금계산서를 발행할 수 없다. 광고 집행이 별도로 차단되지는 않는다. 환불 계좌(④)는 선택 사항으로, 환불 신청 전까지만 등록하면 된다.

변경 정책

🔒
환불 계좌 변경도 계정 책임자 승인 필수

변경 시 동일한 승인 절차를 반복한다. 변경 이벤트 발생 시 계정 책임자에게 이메일 + SMS로 즉시 통보. 본인이 요청하지 않은 경우 즉시 고객센터에 연락하도록 안내.

환불 계좌 변경 이벤트는 보안상 중요도가 높아 SMS 알림이 반드시 필요하다. 현재 SMS 발송 인프라 미구현 상태로 신규 개발이 요구된다. (ADM팀 + 인프라팀 협의 필요)

항목내용

발행 시기전월 소진분 → 익월 1~10일

위임발행 구조

대행사 앞으로 위임발행 선택 시, 버즈빌이 대행사 사업자등록번호로 세금계산서를 발행한다. 대행사가 이를 계정 책임자에게 재청구하거나 자사 비용으로 처리한다. 위임발행의 세법 적합성은 법무팀 확인 필요.
