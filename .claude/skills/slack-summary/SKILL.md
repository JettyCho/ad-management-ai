# 슬랙 채널 대화 요약 및 분석

## 입력값 파싱

전달받은 인자: `$ARGUMENTS`

사용자가 자연어로 자유롭게 입력한 내용에서 **채널**과 **기간**을 스스로 판단하여 추출한다.

**채널 추출:**
- 입력에서 채널명으로 보이는 단어를 추출한다
- `#` 기호가 있으면 제거하고 처리
- 여러 채널이 언급되면 모두 포함
- 예: "general이랑 dev-backend 채널", "#qa 채널 좀 봐줘", "광고관리 관련 채널"

**기간 추출:**
- 자연어 표현을 날짜 범위로 변환한다
- 예: "최근 3일", "이번 주", "지난주", "어제", "3월 첫째 주", "최근 한 달"
- "3d", "7d" 같은 축약 표현도 지원
- `YYYY-MM-DD~YYYY-MM-DD` 형식도 지원
- 오늘 날짜는 `$TODAY` 기준으로 계산

**기본값:**
- 채널이 특정되지 않으면 채널 `ad-management` 사용
- 기간이 특정되지 않으면 최근 7일 적용
- 인자가 아무것도 없으면 채널 `ad-management`, 기간 최근 7일로 실행

## 채널 ID 확인

`team/README.md`의 슬랙채널 섹션에 채널명과 ID가 정리되어 있다. 여기서 먼저 찾는다.
목록에 없는 채널은 `mcp__slack__slack_search_channels`로 검색한다.

---

## 1단계: 데이터 수집

Slack MCP 도구를 사용하여 데이터를 수집한다.

1. **채널 메시지 수집**: `mcp__slack__slack_read_channel`로 채널 히스토리를 조회한다.
   - `oldest`/`latest` 파라미터로 기간을 필터링한다 (Unix timestamp)
   - 한 번에 최대 100건이므로, 메시지가 많으면 `cursor`로 페이지네이션한다

2. **스레드 수집**: 답글이 있는 메시지는 `mcp__slack__slack_read_thread`로 스레드 전체를 조회한다.
   - `message_ts` 파라미터에 부모 메시지의 timestamp를 전달한다

3. 여러 채널이면 채널별로 반복 수행

수집 시 주의사항:
- 봇 메시지, 자동화 알림은 분석에서 제외
- 리액션이 많거나 스레드가 긴 메시지는 중요도가 높은 것으로 표시
- 동일 주제가 여러 날에 걸쳐 논의된 경우 하나의 흐름으로 묶기

---

## 2단계: 1차 분석 및 요약

수집된 내용을 바탕으로 채널별로 아래를 파악한다:

- 주요 대화 주제들 (무엇에 대해 이야기했는가)
- 언급된 작업/과제/프로젝트
- 제기된 문제나 이슈
- 내려진 결정 사항
- 팀원들의 전반적인 분위기나 우선순위

**주요 대화 주제**
- 이 기간 동안 주로 어떤 주제들이 오갔는지 불릿으로 정리

**진행 중인 과제 / 작업**
- 현재 누군가 담당하거나 진행 중인 것들
- 담당자가 언급됐다면 함께 표기

**해결된 문제 ✅**
- 이슈가 제기됐고 결론/해결책이 나온 것들
- 어떻게 해결됐는지 간략히 설명

**중요했던 이슈 ⚠️**
- 리액션이 많거나, 스레드가 길거나, 여러 명이 참여한 이슈
- 왜 중요했는지 맥락 포함

**아직 미해결인 문제 🔴**
- 이슈가 제기됐지만 결론이 나지 않은 것들
- 논의가 흐지부지 끝났거나 아무도 결론을 내리지 않은 것
- 누가 결론을 내려야 할지 제안 가능하면 포함
